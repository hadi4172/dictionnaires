Tri par insertion ↓
............................. 
    public static void insertionSort(int[] array) {
        for (int i = 1; i < array.length; i++) {
            int current = array[i];
            int j = i - 1;
            while(j >= 0 && current < array[j]) {
                array[j+1] = array[j];
                j--;
            }
            array[j+1] = current;
        }
    }

### Layouts
Appliquer un Layout : new JPanel(new GridLayout(args));     //ou utiliser .setLayout(); sur un JComponent
FlowLayout : Les éléments sont disposés de gauche à droite, de haut en bas. Pas d'args. Layout par défaut du JPanel.
BorderLayout : Positions Nord/Est/Sud/Ouest/Centre. Pas d'args. Layout par défaut du JFrame. Exemple : pnlPrincipal.add(btnUn, BorderLayout.NORTH);
GridLayout : Élements disposés dans un tableau de lignes et colonnes. ajoutés de g à d, de h en b. args : (lignes, colonnes) //mettre 0 à un des args rend la gestion automatique.
BoxLayout : Élements disposés selon axe horizontal ou vertical. args: (panneauDeDestination, BoxLayout.X_AXIS / BoxLayout.Y_AXIS)
CardLayout : Permet d'avoir plusieurs couches de Panel et afficher celle que l'on veut quand on veut. exemple : https://bityl.co/4X1h

Ajouter un écart entre les lignes/colonnes de la GridLayout : gl.setVgap(5)/.setHgap(5); //Cinq pixels d'espace

Forcer la mise à jour de l'affichage swing : monPanneau.revalidate(); monPanneau.repaint();
Taille automatique JFrame : pack(); //dans le constructeur au lieu du setSize()  //il faut mettre tout les autres composants avant de faire pack()

Sérialisation
------------------------------
Ouverture d’un fichier binaire en lecture : DataInputStream fichier = new DataInputStream(new FileInputStream("fichier1.bin"));
Ouverture d’un fichier binaire en écriture : DataOutputStream fichier = new DataOutputStream(new FileOutputStream("fichier2.bin"));
Ouverture d’un fichier binaire en mode append : DataOutputStream fichierSortie = new DataOutputStream(new FileOutputStream("fichier2.data", true));

Écrire une donnée string : fichier.writeUTF("bonjour");
Écrire une donnée d'un autre type primitif : fichier.writeInt|writeLong|writeFloat...(4);

Pour lire les données : fichier.readUTF|readInt|readDouble...();        //le type de lecture doit se faire dans le même ordre que le type d'écriture (String pour String , int pour int...)

Lecture et Écriture de données dans un fichier binaire ↓
................
    try {
        DataOutputStream ecrivain;
        ecrivain = new DataOutputStream(new FileOutputStream("fichier.data"));
        ecrivain.writeUTF("bonjour");
        ecrivain.writeInt(3);
        ecrivain.writeLong(100000);
        ecrivain.close();

        DataInputStream lecteur = lecteur = new DataInputStream(new FileInputStream("Fichier.data"));
        System.out.println(lecteur.readUTF());
        System.out.println(lecteur.readInt());
        System.out.println(lecteur.readLong());
        lecteur.close();
    } catch (FileNotFoundException ex) {
        System.out.println("Fichier inexistant.");
    } catch (IOException ex) {
        System.out.println("Erreur d'accès au fichier.");
    }
................


writeObject(monObjet) : écrit l'objet dans le fichier binaire spécifié ;
readObject() : Cette méthode permet de "lire" dans le fichier les états sauvegardés de l'objet.
flush() : vide le cache et force l'écriture dans le fichier binaire
close() : Permet de fermer le flux.
IOException : Permet de lever l’exception s’il y a problème avec le fichier.
ClassNotFoundException : Permet de lever l’exception si l'objet est transtypé vers une classe qui n'existe plus ou pas au moment de l'exécution.
StreamCorruptedException : Permet de lever l’exception si le fichier a été corrompu (par exemple en le modifiant avec un éditeur).

Consumer<T>      : 1 paramètre et 0 sortie                          //il existe aussi BiConsumer<T,​U>   (2 paramètres et 0 sortie)
